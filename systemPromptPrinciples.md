# General  
The topic of **System Prompts** is quite extensive and has a significant impact on the behavior of chat models.  
Therefore, it is important to explore the topic in depth and to build up an expert within the team.

# Sources  
Below are sources related to the topic of System Prompts.

## Books  
- The book **Mastering System Prompts for LLMs** is very comprehensive and covers all aspects of working with system prompts.

# Template  
- A template for the organization and team that defines the basic structure of a system prompt template can be found in `systemPromptTemplate.yaml`.  
- The template should be created in a format such as JSON or YAML.  
- The template should follow formatting rules for topics like `topic:`, placeholders like `{{}}`, documentation using `#`, etc.  
- The template should be version-controlled.

# Topics  
- It should be clearly defined which topics are to be included in the template.  
- There should be clear rules on how the wording under each topic should be structured — for example, how prohibitions are to be precisely formulated.  
- Best practices should be developed for such wording and illustrated in the template as examples (see appendix *Tools*).

# Instance  
- For a specific application, e.g., an insurance chatbot, the template is instantiated by filling it with specific content (`systemPrompt.yaml`).  
- The initial content could also be generated by a generative AI (LLM), which can then be further refined through a defined process.  
- Alternatively, a sample (e.g., from *Awesome ChatGPT Prompts*) could be used as a starting point and refined.

# Compiler  
- A compiler should turn the template instance into a readable prompt text.  
- A prompt compiler can be written in Python, JavaScript, or even as a simple Bash script.  
- Typically, steps like formatting cleanup (e.g., removing `#`), filling in placeholders like `{{}}`, and saving to a file (e.g., `systemPrompt.sh`) are necessary.

# Usage  
- The resulting prompt (e.g., `systemPrompt.st`) should be filled with domain-specific, contextual content.
